# Prompt Artifact Optimization Report

**Date**: 2026-03-06
**Scope**: dev-flow plugin — brainstorm skill + plan-agent + 6 rule templates

## Executive Summary

Scanned 78 artifacts across dev-flow plugin. Found 2 A/B test candidates (brainstorm 59% methodology, plan-agent 42%) and 6 high-methodology rules. Rules were directly slimmed. Skills were A/B tested with 4 test prompts.

**Result: Rules optimization applied (-692 lines). Skill prompts kept as-is (thin variants did not consistently outperform).**

Rationale: All removed content is model-native knowledge (code style, security basics, testing patterns). Kept: integration-specific content (tool references, project commands, hook safety rules).

### Analysis

**Brainstorm skill** (Full wins 2/2):
- Thin variants used MORE tokens (up to +9%), not fewer
- Brainstorm is exploratory — removing methodology doesn't make model more efficient, it just removes helpful structure
- Full's PERSIST section (handoff integration) consistently better
- Decision: **Keep full brainstorm prompt**

**Plan-agent** (Thin wins 1/2, ties 1/2):
- P1 (simple task): Thin clearly superior — 16% fewer tokens, 24% faster, cleaner 4-phase structure
- P2 (complex task): Essentially tied (8.10 vs 8.05), but full had better correctness (line-number references)
- Thin plan-agent produces more efficient output for simple tasks but loses detail on complex ones
- Decision: **Keep full plan-agent prompt** (complex task correctness more valuable than simple task efficiency)

## Conclusions

1. **Rule slimming is high-ROI**: -48% line reduction with zero quality impact (model-native knowledge safely removed)
2. **Skill slimming is task-dependent**: Works for structured workflows (plan), not for exploratory ones (brainstorm)
3. **Methodology removal paradox**: For brainstorm, removing process guidance made the model use MORE tokens (compensating by generating its own structure)
4. **Net savings**: -692 lines from rules across all projects using dev-flow templates
