## ⚠️ 优化建议

### P1 - 重要（建议处理）

#### 1. 原生 Auto-memory (v2.1.59) — 需要确认冲突
**新功能**: Claude Code 内置了 `/memory` 管理的自动记忆系统，会自动保存对话中的有用信息。

**当前状态**: 用户已有完整的自定义 auto-memory 系统：
- `~/.claude/projects/-Users-lazyman-.../memory/MEMORY.md`
- `~/.claude/rules/memory-rules.md` 定义写入规则

**风险**: 原生 auto-memory 可能与自定义系统产生冲突或冗余。
- 原生系统会把内容写到哪里？与 `memory/MEMORY.md` 是否隔离？
- 用户是否希望两套系统共存？

**建议**: 用 `/memory` 命令查看原生系统当前状态，决定是否需要显式关闭。

---

#### 2. `/simplify` 内置命令 (v2.1.63) — 可能替代插件
**新功能**: `/simplify` 现在是 Claude Code 内置的 slash command（bundled）。

**当前状态**: 已安装 `code-simplifier@claude-plugins-official` 插件。

**建议**: 检查内置 `/simplify` 和插件的功能是否重复，如重复可考虑卸载插件：
```bash
claude plugins remove code-simplifier@claude-plugins-official
```

---

#### 3. `WorktreeCreate` / `WorktreeRemove` hooks (v2.1.50) — 新 hook 类型
**新功能**: 新增两个 hook 事件，在 worktree 创建/移除时触发，适合自定义 VCS 设置。

**当前状态**: 未配置这两种 hook。

**适用场景**: 用户频繁使用 worktree，可以：
- `WorktreeCreate`: 自动初始化 worktree 环境（如复制 .env.local）
- `WorktreeRemove`: 清理 worktree 特定资源

**建议**: 目前可选，如有 worktree 初始化需求再添加。

#### 5. `isolation: worktree` in agent definitions (v2.1.50) — 代理 frontmatter
**新功能**: Agent 定义文件可以加 `isolation: worktree` 字段，代理自动在隔离的 git worktree 中运行。

**当前状态**: Agent frontmatter 目前不含此字段，需要在 `Agent()` 调用时传参数。

**建议**: 对于 code-reviewer、implement-agent 等需要隔离的代理，可考虑在 `.md` 文件中加 `isolation: worktree`。

示例（`agents/code-reviewer.md`）:
```yaml
---
name: code-reviewer
description: ...
isolation: worktree
---
```

### P3 - 了解即可

#### 8. BashTool 不再默认使用 login shell (v2.1.51)
**变更**: BashTool 默认跳过 login shell 初始化（不再加 `-l` 标志）。

**影响**: 依赖 `.zprofile`/`.bash_profile` 的工具（如 nvm、rbenv）可能在 Bash hook 中不可用。

**建议**: 监控 hooks 中是否有依赖 login shell 的命令失败。如需 login shell，可在命令中手动加 `bash -l -c "..."`.
