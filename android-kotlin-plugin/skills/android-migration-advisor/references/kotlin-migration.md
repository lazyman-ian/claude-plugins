# Kotlin Migration Reference

Patterns for converting Java codebases to idiomatic Kotlin.

## J2K Converter Best Practices

Android Studio's Java-to-Kotlin converter (`Code → Convert Java File to Kotlin File`) handles ~80% automatically. Manual review is required for:

| Scenario | Problem | Fix |
|----------|---------|-----|
| Nullable platform types | `String!` — unknown nullability | Add explicit `String?` or `String` |
| Static inner classes | Converted to nested (wrong) | Add `inner` or keep as top-level |
| Singleton patterns | `getInstance()` → companion | Verify thread safety with `@Volatile` |
| Anonymous classes | Converted to objects or lambdas | Verify capture semantics |
| Builder patterns | Stays verbose | Replace with named parameters |

```kotlin
// Before (Java-style after conversion) — fix this:
val result = Builder()
    .setName("Alice")
    .setAge(30)
    .build()

// After (idiomatic Kotlin):
data class User(val name: String, val age: Int)
val result = User(name = "Alice", age = 30)
```

## Null Safety at Java/Kotlin Boundary

Java methods return platform types (`T!`) — treat as potentially null:

```kotlin
// Java method: String getName() — returns String! in Kotlin
val name: String = javaObject.name  // Compiles, may crash at runtime

// Safe approach — explicit nullable:
val name: String? = javaObject.name
val display = name ?: "Unknown"

// Or use requireNotNull at trusted boundaries:
val name: String = requireNotNull(javaObject.name) { "name must not be null" }
```

For Java collections, apply defensive copying:

```kotlin
// Java List<String> becomes List<String!> — elements may be null
val javaList: List<String> = javaService.getItems()
val safeList: List<String> = javaList.filterNotNull()
```

## Interop Annotations

Use these when Kotlin code must be called from Java:

```kotlin
class UserRepository {
    companion object {
        // @JvmStatic: java can call UserRepository.getInstance() instead of Companion.getInstance()
        @JvmStatic
        fun getInstance(): UserRepository = UserRepository()

        // @JvmField: exposes as field, not getter/setter
        @JvmField
        val DEFAULT_TIMEOUT = 30_000L
    }

    // @JvmOverloads: generates Java overloads for default parameters
    @JvmOverloads
    fun fetchUser(id: String, includeDetails: Boolean = false): User { ... }
}
```

```kotlin
// @JvmName: rename generated bytecode to avoid clashes
@JvmName("filterActiveUsers")
fun List<User>.filterActive(): List<User> = filter { it.isActive }
```

## SAM Conversions for Java Interfaces

```kotlin
// Java interface: interface OnClickListener { void onClick(View v); }
// Kotlin lambda converts automatically:
button.setOnClickListener { view -> handleClick(view) }
// Or even simpler:
button.setOnClickListener(::handleClick)

// Custom Java functional interface in Kotlin:
val callback = Runnable { doWork() }
executor.submit(callback)
```

## Extension Functions for Java APIs

Wrap verbose Java APIs with extension functions:

```kotlin
// Before:
val calendar = Calendar.getInstance()
calendar.set(Calendar.YEAR, 2024)
calendar.set(Calendar.MONTH, Calendar.JANUARY)
val date = calendar.time

// After — extension on Calendar:
fun Calendar.setYearMonth(year: Int, month: Int): Calendar = apply {
    set(Calendar.YEAR, year)
    set(Calendar.MONTH, month)
}
val date = Calendar.getInstance().setYearMonth(2024, Calendar.JANUARY).time

// Extension on cursor (common Java API):
fun Cursor.getString(columnName: String): String =
    getString(getColumnIndexOrThrow(columnName))
```

## Companion Object vs Static

```kotlin
// Java static utility class:
public class MathUtils {
    public static int add(int a, int b) { return a + b; }
}

// Kotlin option 1 — companion object (use when class has instances too):
class Calculator {
    companion object {
        fun add(a: Int, b: Int) = a + b
    }
    fun multiply(a: Int, b: Int) = a * b
}

// Kotlin option 2 — top-level function (preferred for pure utilities):
// MathUtils.kt
fun add(a: Int, b: Int) = a + b
```

## Sealed Classes Replacing Java Patterns

```kotlin
// Java: enum with data — limited to constants
public enum class Status { LOADING, SUCCESS, ERROR }

// Kotlin sealed class — each subtype can carry data:
sealed class UiState<out T> {
    object Loading : UiState<Nothing>()
    data class Success<T>(val data: T) : UiState<T>()
    data class Error(val message: String, val cause: Throwable? = null) : UiState<Nothing>()
}

// Exhaustive when — compiler enforces all cases:
when (val state = viewModel.uiState.value) {
    is UiState.Loading -> showSpinner()
    is UiState.Success -> showData(state.data)
    is UiState.Error -> showError(state.message)
}
```

## Data Classes Replacing Java POJOs

```kotlin
// Java POJO (100+ lines with equals/hashCode/toString/builder):
public class Product { ... }

// Kotlin data class (1 line):
data class Product(
    val id: String,
    val name: String,
    val price: Double,
    val category: Category = Category.GENERAL
)

// copy() for immutable updates (replaces Builder pattern):
val discounted = product.copy(price = product.price * 0.9)

// Destructuring:
val (id, name, price) = product
```

## Coroutine Integration With Java Callbacks

```kotlin
// Wrap Java callback API in suspendCoroutine/suspendCancellableCoroutine:
suspend fun LocationManager.getLastLocationSuspend(): Location? =
    suspendCancellableCoroutine { cont ->
        val listener = LocationListener { location -> cont.resume(location) }
        requestSingleUpdate(GPS_PROVIDER, listener, Looper.getMainLooper())
        cont.invokeOnCancellation { removeUpdates(listener) }
    }

// Usage:
val location = locationManager.getLastLocationSuspend()
```

```kotlin
// Convert ListenableFuture (Guava/Retrofit) to suspend:
suspend fun <T> ListenableFuture<T>.await(): T =
    suspendCancellableCoroutine { cont ->
        addListener(
            { cont.resumeWith(runCatching { get() }) },
            MoreExecutors.directExecutor()
        )
        cont.invokeOnCancellation { cancel(true) }
    }
```

## Gradle Setup

```kotlin
// build.gradle.kts (module-level)
plugins {
    id("com.android.application")  // or library
    id("org.jetbrains.kotlin.android")
    id("com.google.devtools.ksp")  // prefer KSP over kapt
}

android {
    kotlinOptions {
        jvmTarget = "17"
        freeCompilerArgs += listOf(
            "-opt-in=kotlin.RequiresOptIn",
            "-opt-in=kotlinx.coroutines.ExperimentalCoroutinesApi"
        )
    }
}
```

## Large-Scale Automation (Meta's Kotlinator Approach)

For codebases with 1,000+ Java files, manual conversion is impractical. Key automation steps:

1. **Batch J2K conversion**: Convert modules bottom-up (no dependents first)
2. **Automated null annotation inference**: Use tools like `NullAway` to add `@Nullable`/`@NonNull` before conversion, improving J2K output quality
3. **Post-conversion linting**: Run `detekt` and `ktlint --format` automatically after each batch
4. **Incremental CI gating**: Add Kotlin compilation check to CI; block new Java files in converted modules

```bash
# Convert all Java files in a module (run from module root):
find . -name "*.java" -not -path "*/build/*" | while read f; do
    # Android Studio CLI conversion (requires AS installation)
    # In practice, use IDE macro or scripted J2K invocation
    echo "Converting: $f"
done
```

**Realistic timeline**: 500 Java files ≈ 4-6 weeks with 2 engineers (conversion + review + test fix).
